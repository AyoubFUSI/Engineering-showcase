Computational Fluid Dynamics: 1D Flow Modeling in Finned Channels

Objective: To establish a computational model for the velocity profile of a liquid coolant flowing between the narrow channels of a finned thermal unit, bridging analytical physics and discrete numerical methods.

1. Physical Scenario and Governing Assumptions

We model an incompressible Newtonian fluid flowing steadily through a narrow gap of width $2h$ between two parallel cooling fins. To streamline the mathematical model for 1D computational analysis, we apply the following standard assumptions:

  1. Steady State: Flow properties are time-independent ($\partial / \partial t = 0$).
  2. Fully Developed Laminar Flow: The velocity profile remains constant along the flow direction ($\partial u / \partial x = 0$).
  3. 1D Flow: Fluid motion is restricted to the x-direction ($v = 0, w = 0$).

2. Governing Equation (Navier-Stokes Reduction)

The full Navier-Stokes momentum equation for the x-direction simplifies significantly under these assumptions. Convective terms and gravity become negligible, leaving a balance between the pressure gradient and viscous shear forces:

$$ 0 = -\frac{dP}{dx} + \mu \frac{d^2 u}{dy^2} $$

Rearranging isolates the second derivative of velocity:

$$ \frac{d^2 u}{dy^2} = \frac{1}{\mu} \frac{dP}{dx} $$

(Where $\mu$ is dynamic viscosity, $u$ is velocity, and $dP/dx$ is the pressure drop).

3. The Analytical Solution

Integrating the simplified differential equation twice with respect to $y$ yields:

$$ u(y) = \frac{1}{2\mu} \frac{dP}{dx} y^2 + C_1 y + C_2 $$

Applying the no-slip boundary conditions at the fin walls ($u(h) = 0$ and $u(-h) = 0$) allows us to solve for the constants, resulting in the exact parabolic velocity profile:

$$ u(y) = -\frac{1}{2\mu} \frac{dP}{dx} (h^2 - y^2) $$

4. Numerical Discretization for the Solver

To implement this computationally, we discretize the fluid domain into a grid of nodes with spacing $\Delta y$. Using the Central Difference scheme of the Finite Difference Method, the second derivative is approximated as:

$$ \frac{d^2 u}{dy^2} \approx \frac{u_{i+1} - 2u_i + u_{i-1}}{\Delta y^2} $$

Substituting this into the Navier-Stokes reduction yields the algebraic equation for each node:

$$ u_{i-1} - 2u_i + u_{i+1} = \frac{\Delta y^2}{\mu} \frac{dP}{dx} $$

When applied across $n$ nodes, this forms a Tridiagonal Matrix system ($Ax = b$), which can be solved with $O(n)$ efficiency using the Thomas Algorithm.

5. Implementation (Thomas Algorithm / TDMA)

The following script demonstrates the solver implementation. The underlying matrix operations are straightforward and can be easily migrated to C/C++ or MATLAB/Simulink for high-performance simulation pipelines.

import numpy as np

def solve_tdma(a, b, c, d):
    """
    Solves a tridiagonal matrix system Ax = d using the Thomas Algorithm.
    """
    n = len(d)
    w = np.zeros(n-1, float)
    g = np.zeros(n, float)
    u = np.zeros(n, float)
    
    # Forward sweep
    w[0] = c[0] / b[0]
    g[0] = d[0] / b[0]
    
    for i in range(1, n-1):
        w[i] = c[i] / (b[i] - a[i-1] * w[i-1])
    for i in range(1, n):
        g[i] = (d[i] - a[i-1] * g[i-1]) / (b[i] - a[i-1] * w[i-1])
        
    # Back substitution
    u[n-1] = g[n-1]
    for i in range(n-1, 0, -1):
        u[i-1] = g[i-1] - w[i-1] * u[i]
        
    return u

# --- Physical Setup ---
nodes = 50           # Number of discrete grid points
h = 0.005            # Half-width of the channel (5 mm)
mu = 0.001           # Dynamic viscosity (Pa.s)
dp_dx = -50.0        # Pressure gradient (Pa/m)

# Create spatial grid
y = np.linspace(-h, h, nodes)
dy = y[1] - y[0]

# --- Matrix Construction ---
a = np.ones(nodes - 1)                                
b = -2.0 * np.ones(nodes)                             
c = np.ones(nodes - 1)                                
d = np.ones(nodes) * (dy**2 / mu) * dp_dx             

# Enforce Boundary Conditions (No-slip at walls)
b[0] = 1.0; c[0] = 0.0; d[0] = 0.0                    
b[-1] = 1.0; a[-1] = 0.0; d[-1] = 0.0                 

# --- Execution ---
velocity_profile = solve_tdma(a, b, c, d)

Conclusion: By wrapping this highly efficient computational solver into an automation loop or larger architecture, it becomes possible to rapidly simulate thousands of different fin widths or pressure gradients to optimize system geometries for thermal performance.
